闭包和作用域

产生闭包的两种表现

  • 函数作为参数被传递

  • 函数作为返回值被返回

形成闭包，执行闭包里的函数，那么所使用的变量将从函数定义的域向上查找，而不是在执行函数所在的域。把形成的闭包想象成一颗气泡，会把最初的域包裹进来并隔离闭包外的变量。

this

  • 使用场景
    • 作为普通函数
    • 使用 call apply bind
    • 作为对象被调用
    • 在 class 方法中调用
    • 箭头函数

和闭包相反，this 取的值由函数执行时域确认，而不是函数定义时域。

  • this 导致非凡的注意点
    • 调用全局方法时，指向 window
    • 箭头函数作为函数无法通过 apply, call, bind 方法指定其 this，箭头函数没有自己的 this，但是会继承上层作用域的 this
