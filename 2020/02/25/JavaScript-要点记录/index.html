闭包和作用域

产生闭包的两种表现

  • 函数作为参数被传递

  • 函数作为返回值被返回

形成闭包，执行闭包里的函数，那么所使用的变量将从函数定义的域向上查找，而不是在执行函数所在的域。把形成的闭包想象成一颗气泡，会把最初的域包裹进来并隔离闭包外的变量。

this

  • 使用场景
    • 作为普通函数
    • 使用 call apply bind
    • 作为对象被调用
    • 在 class 方法中调用
    • 箭头函数

和闭包相反，this 取的值由函数执行时域确认，而不是函数定义时域。

  • this 导致非凡的注意点
    • 调用全局方法时，指向 window
    • 箭头函数作为函数无法通过 apply, call, bind 方法指定其 this，箭头函数没有自己的 this，但是会继承上层作用域的 this

Map

对象类型数据的 key 必须是字符串。当使用非字符串类型如 HTML 元素作为 key 时，其值被 toString 方法转为字符串类型。

向 Map 添加成员，作为构造函数，可以接受数组作为参数。

Map 的键和内存地址绑定，这能解决了同名属性碰撞（clash）的问题。

两个值严格相等，Map 则视为同一个键，NaN除外。如 0, -0, +0 被视为一个键。

缓存

expires

HTTP1.0 字段，设置具体的时间值作为缓存资源过期时间。

Cache-Control

HTTP1.1 字段，设置相对时间控制缓存过期时间。cache-control: max-age=315360000表示过期时间一年。

Cache-Control 字段的字段：public, private, no-cache, no-store, max-age=, s-maxage=, max-stale[=]

Last-Modified / If-Modified-Since

第一次请求服务器返回 Last-Modefied 后浏览器保存为 If-Modified-Since，之后浏览器每次发出请求，服务器对比资源最后修改时间来确定返回 304 或 200。

Etag / If-None-Match

使用 Hash 判断资源是否被修改，因此不受秒级准确度限制，更精准。

自定义事件

  • 为什么要使用自定义事件？松耦合，当代码触发匹配条件不需要指定关于条件的细节代码。
  • 为什么不直接调用全局函数？调用的代码为函数，需要定义一个固定名称，并且使用共享函数的页面均需要使用这样的一个函数。

Module

  • ES6 模块输出的是值的引用

Node.js 加载

  • v13.2 开始，Node.js 默认打开了 ES6 模块支持
  • .mjs文件总是以 ES6 模块加载；.cjs文件总是以 CommonJS 模块加载；.js文件的加载取决于package.json里面type字段的设置，如果没有type字段或者type字段为commonjs，则文件会被解释成 CommonJS 模块
  • 比较简单的模块，可以只使用main字段，指定模块加载的入口文件，然后，import命令就可以加载这个模块，ryf-main 字段
  • exports字段的优先级高于main字段，ryf-export 字段

循环加载

ES6 的循环加载

  • ES6 模块是动态引用，如果使用import从一个模块加载变量（即import foo from 'foo'），那些变量不会被缓存，而是成为一个指向被加载模块的引用，需要开发者自己保证，真正取值的时候能够取到值
